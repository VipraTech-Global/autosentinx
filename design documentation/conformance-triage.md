- **Landing (`app/page.tsx`) fully rewritten** — `GridCanvas`, `MarketingNav`, glass console mockup, FRAMEWORKS/FEATURES/STEPS/METRICS/FAQ, multi-column footer. None of our landing DOM survives. `components/minimal-bar.tsx` is no longer used by the landing (replaced by `MarketingNav`).

- **Run config (`app/new/page.tsx`) rewired to real API** — `startScan` → `approveScan`; added an `approving` phase. Still an inline Card view-swap, still happy-path, still old copy.
- **Run pages (`app/runs/[id]/**`) rewritten for live data** (we never edited any `runs/*` page — our diff has zero hunks there; see rows 17/23). They `useRun(...)` poll and consume our shared components (`badges`, `evidence`, `findings`, `observations-table`).
- **`lib/api.ts` already routes live observations through `deriveOutcome`** (single source of truth) for the backend path.
- **Cross-cutting component/lib files we fixed are otherwise UNCHANGED from our base** in main (still carry the original bugs): `outcome.ts`, `score.ts`, `ui.tsx`, `theme-toggle.tsx`, `disclosure.tsx`, `badges.tsx`, `findings.tsx`, `logo.tsx`, `run-tabs.tsx`, `observations-table.tsx`, `evidence.tsx` (logic/bugs unchanged, but its brand strings were swept to "AutoSentinx" in main — see row 18), `processing-view.tsx` (this one changed — see below), `mock/run.ts`.

  - **Processing poll-vs-timer (#14)** — superseded; main's `ProcessingView` already polls `/runs/{id}` via `useRun`. (The *rest* of our processing work is still needed — TWEAK #15.)
